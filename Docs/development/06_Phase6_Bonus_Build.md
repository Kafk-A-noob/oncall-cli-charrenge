# Phase 6: (Bonus) 実行可能バイナリへのコンパイル

## 概要

本プロジェクトでは、Bunの強力なコンパイル機能を用いて、CLIツールを単一の実行可能ファイル（`.exe` 等）にビルドすることができます。
この手順を行うことで、ツールを利用する別のエンジニア（または審査員）のPCに `Bun` や `Node.js` がインストールされていなくても、コマンド一発でツールを起動できる「究極のZero-setup体験」を提供できます。

## 作業手順

### 1. バイナリのビルド実行

ターミナルで以下のコマンドを実行します。

```bash
bun build --compile src/cli/index.ts --outfile oncall
```

- `--compile`: プロジェクト内の全コード、依存ライブラリ、そしてBunの実行エンジン自体をガッチリと1つのファイルにパッキングする魔法のオプションです。
- `--outfile oncall`: 出力される実行ファイルの名前を指定します。Windows環境で実行した場合は自動的に `oncall.exe` が生成されます。

### 2. 実行テスト

ビルドが成功すると、プロジェクトの直下に `oncall`（または `oncall.exe`）が生成されます。
以降は `bun run src/cli/...` といった煩わしい呼び出しや、余計な内部コマンドの印字(`$ bun run...`)を避けて、直接以下のように呼び出せます。

```bash
# ヘルプの表示
./oncall -h

# 障害状況の確認
./oncall "Discordの状況教えて"
```

> **メリット:**
> これにより、ツール利用者は `-h` で表示された通りのクリーンな体験を得られます。（CLIのUXとしては現行最高峰の配布形態と言えます）
