# Oncall CLI Agent 結合テスト（E2E）項目書

本ドキュメントは、ツールのすべての機能が要件通りに動作するかを確認するためのテストシナリオです。

## テスト前準備

- [ ] カレントディレクトリが `oncall-cli-charrenge` (内側のプロジェクトルート) になっていること
- [ ] `.env` または `.env.example` に有効な Gemini API キーが設定されていること
- [ ] 依存パッケージがインストールされていること (`bun install`)

---

## 1. ヘルプとエラーハンドリングのテスト (CLI層の責務)

### 1-1. 引数なし起動

- **実行コマンド:** `bun run start` (または `bun run src/cli/index.ts`)

- **期待される結果:**
  - `エラー：調査したいサービスを引数で渡してください` というメッセージが出力されること。
  - 実行例（使い方）が提示されること。
  - プロセスが異常終了（ステータス `1`）で完了すること。

    結果

    ``` text
    $ bun run src/cli/index.ts
    エラー：調査したいサービスを引数で渡してください
    例: bun run start "Discordの状況を調査して"
    error: script "start" exited with code 1
    ```

    と表示された。
    期待通りの内容は出ているものの、`$ bun run src/cli/index.ts`の記載は見えるものなのだろうか？要検証
    → 【AI解説】パッケージマネージャー（bun / npm 等）の仕様で、実行した裏コマンドがエコー印字される現象です。動作として100%正常です。

### 1-2. ヘルプ起動

- **実行コマンド:** `bun run start -h` (または `--help`, `help`)

- **期待される結果:**
  - ツールのタイトルと使い方、対応しているサービスの一覧が綺麗に表示されること。
  - プロセスが正常終了（ステータス `0`）で完了すること。

    結果

    ``` text
    $ bun run src/cli/index.ts -h

      =========================================
      Oncall CLI Agent 初動調査AIツール

      使い方: bun run start "<聞きたいこと>"
      対応サービス:
      - GitHub
      - Discord
      - Cloudflare
      実行例:
      bun run start "DiscordとGitHubの状況を調査して"
      =========================================
    ```

    と表示された。
    期待通りの内容は出ているものの、`$ bun run src/cli/index.ts -h`の記載は見えるものなのだろうか？1に続いて検証するとともに、結局`bun run start`の後にhelp呼び出しをかけなくてはいけないのか？。
    今回は先方の環境に影響させないために致し方ないかも含め検討。
    → 【AI見解】「余計なエコーを消したい」「コマンドが少し不格好」という懸念については、Phase 6で言及した「単一バイナリ化（bun build --compile）」を行うことで全て解決します。バイナリを渡せば `./oncall -h` だけで済み、「先方のPC環境への影響（汚染）をゼロにする気遣い」として提出の完成形態に昇華されます。
